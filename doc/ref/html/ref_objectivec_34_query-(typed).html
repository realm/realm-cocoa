Query (typed) Class

Query objects are used to build up queries. A query is tied to a Table (typed). The typed query class is used in conjunction with Table (typed).

Conditions is added to a query through a fluent interface. This means that the methods representing the conditions will return the query object.

Conditions are split in two parts: the column name and a method for each predicate. Conditions are folded with logical-and if not stated otherwise.

Once the query is ready, it is possible to reuse by multiple calls to the action methods. Moreover, it is possible extend the query with new conditions after applying an action method and thereby implement a drill-down. It is important to stress that no data is cached, and the new queries cannot benefit from previous queries.

Alternatively, a query can result in a TableView (typed) which can be further queried.

Method Summary:
Conditions
equal() Equal to.
notEqual() Not equal to.
greater() Greater than.
greaterEqual() Greater than or equal to.
less() Less than.
lessEqual() Less than or equal to.
between() Belongs to an interval.
beginsWith() Substring match at the begining of the attribute.
endsWith() Substring match at the end at the attribute.
contains() Substring search.
Combiners
group() Start group ("left parenthesis").
endGroup() Stop group ("right parenthesis").
or() Logical-or.
Actions
findAll() Execute query.
findNext() Find next row.
sum() Calculate sum.
count() Number of matching rows.
max() Highest value.
min() Lowest value.
avg() Calculates the average.
remove() Remove rows.
