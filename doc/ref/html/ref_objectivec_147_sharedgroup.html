SharedGroup Class

Database files can be shared across running applications.

When two threads or processes want to access the same database file, they must each create their own instance of SharedGroup.

If the database file does not already exist, it will be created. When multiple threads are involved, it is safe to let the first thread, that gets to it, create the file.

While at least one instance of SharedGroup exists for a specific database file, a lock file will exist too. The lock file will be placed in the same directory as the database file, and its name is derived by adding the suffix '.lock' to the name of the database file.

Processes that share a database file must reside on the same host.

Moreover, the class also provide a transactional interface to TightDB. Transactions are divided into read and write transactions. While read transactions do not modify the table and thereby the group, write transactions do. This division is important to remember as derived objects (tables, views, etc.) are either writable or not.

Method Summary:
Constructor
SharedGroup() Create a shared group.
Write transactions
writeTransaction() Initiate a transaction.
Read transactions
readTransaction() Initiate reading from a shared group.
