constructor

new Table(array $schema)

Instantiates a TightDB table.

Note: Only tables which are part of a group can be serialized to memory or disk.

Parameters:

The table constructor accepts an associative array as argument, where keys are the column names, and values are the column types.

Valid column types: 

"string": column of strings.
"int":    column of integers.
"bool":   column of booleans.
"date":   column of dates.
"binary": column of binary data
"mixed":  column of mixed (heterogeneous) types
array():  column of sub-tables.

Returns:

Table object

Example:

// Create a new table with 4 typed columns
// (where the last one contains sub-tables, which themselves have 2 columns)
$table = new Table(array("name"   => "string",
                         "age"    => "int",
                         "hired"  => "bool",
                         "phones" => array("desc"   => "string",
                                           "number" => "string")));

// Add a single row
$table[] = array("Jim", 34, true, array(array("mobile", "432-424-3242"),
                                        array("work",   "342-432-4234")));

Square Bracket Notation

The square bracket notation can be used for accessing and manipulating elements in a table. As PHP arrays, Tightdb tables have zero based index.

Adding new rows:

$table[] = array $values

New rows can be added at the end of the table by assigning to it with an empty pair of brackets.

Example: 

// Add new rows to the table
$table[] = array("name" => "James", "age" => 20, "hired" => true, "phones" = array());
$table[] = array("John", 20, true, null); // (terse version)

// Adding a row to a sub-table
$table[5]->phones[] = array("school", "434-423-2432") ;
Setting rows:

$table[$row_id] = array $values

The bracket syntax with an index can be used to update an entire row.

Example: 

// Updating an entire row
$table[4] = array("Eric", 50, true, null);
Getting values:

mixed $table[$row_id]->column_name

To access the individual rows of our table, we can use the brackets syntax with an index. This gives us direct access to the values in the row.

Example: 

// Get value
echo $table[5]->name;

// Get value, counting from the back
echo $table[-1]->name;

// You can also access sub-tables directly
echo $table[-1]->phones[0]->number;
Setting values:

$table[$row_id]->column_name = $value

The brackets syntax with a index can also be used for updating specific values in the table.

Example: 

// Changing values
$table[5]->age = 43;
Deleting rows:

unset($table[$row_id]);

A row specified by its index can be deleted using the unset function

Example: 

// Delete a row
unset($table[2]);

find_all()

TableView Table::find_all(string $column_name, mixed $value)

Find all matching values in a specific column.

Parameters:

$column_name: Name of the column,
$value: Value to search for.

Returns:

TableView: A virtual table containing the matching rows. You can interact with the TableView just as a regular table, and all changes to it will propagate to the original table.

Example:
								
// Find all entries with age == 19
$view = $t2->find_all("age", 19);

// Print number of matches
echo "  found ".$view->count()." matches:\n";

// Print all matches (with indexes)
foreach ($view as $ndx => $row) {
	echo "    ".$ndx.": ".$row->name."\n";
}

// Calculate the sum of the matching ages
echo "  total age: ".$view->sum("age")."\n";

// Change the value - will propagate to the table $t2
$view[0]->age++;

query()

TableQuery Table::query()

Create a query object for a table.

Parameters:

N/A

Returns:

TableQuery: Query object

Query interface:

Query objects are created by and associated with table objects. Query is build using a fluent interface (method invocations can be daisy chained). Conditions are by default folded using logical-and. Query objects can be reused, and queries can be repeatly executed by the do() method (resembles the prepare/execute pattern in SQL clients).

The table shows the declared methods on TableQuery objects. All methods except do() returns the query object itself:

$view2 = $table->query()->contains("name", "ill")->do();

$query = $table->query()->gt("age", 19);
$view3 = $query->do();

// Re-run query after change
$table[10]->age = 50;   // update age
$view4 = $query->do();  // $view3 and $view4 are different result set

constructor

new Group()
new Group(string $source, string $filename_or_buffer)

Creates a table group.

Parameters:

No arguments: Creates a new group.

$source: "file" (named file on disk) or "memory" (volatile buffer)

$filename_or_buffer: If source is "file", this should be the path to the file. If "memory" it should be a string containing the serialized group.

Returns:
A group object
Example:

// Create new (empty) Group
$group = new Group();

// Load a group from disk
$group_from_file = new Group("file", "emplyees.tables");

// Load a group from memory
$group_from_mem = new Group("memory", $buffer);
